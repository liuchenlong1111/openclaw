# OpenClaw 本周更新动态 🚀

大家好，这里是 OpenClaw 每周动态，为你带来最新的项目进展。

## 最新版本进展

截至 **2026 年 3 月 29 日**，OpenClaw 项目正保持活跃开发，最近的正式版本是 **2026.3.13**，带来了大量功能更新和问题修复。

## 主要功能更新

### 📱 移动端体验优化

- **Android 聊天设置**：重新设计了聊天设置页面，分组展示设备和媒体部分，刷新了连接和语音选项卡，在移动端提供更紧凑布局
- **iOS 引导页**：新增首次运行欢迎页面，在网关设置前展示，不再自动打开 QR 扫描器，连接步骤展示更清晰的 `/pair qr` 说明
- **Android QR 扫描**：改用 Google Code Scanner，相比传统的 ZXing 流程更加可靠

### 🌐 浏览器自动化增强

- **Chrome DevTools MCP**：新增官方的 Chrome 已登录会话附加模式，支持通过 `chrome://inspect/#remote-debugging` 启用，并提供直达官方设置指南的链接
- **浏览器会话选择**：内置 `profile="user"`（主机已登录浏览器）和 `profile="chrome-relay"`（扩展中转），简化了真实浏览器调用流程
- **批处理自动化**：新增批量动作支持，选择器定位和延迟点击，标准化批量分发流程，感谢 @vincentkoc 的贡献

### 🐳 容器与部署改进

- **时区支持**：新增 `OPENCLAW_TZ` 环境变量，`docker-setup.sh` 可以将网关和 CLI 容器固定到选择的 IANA 时区，感谢 @Lanfei

### 📊 仪表盘优化

- **避免 UI 冻结**：修复了 v2 仪表盘在每个工具结果都重新加载完整聊天历史的问题，工具密集型运行不再触发 UI 冻结/重新渲染风暴
- **文本渲染**：超大纯文本回复现在正常渲染为段落，而不是带封顶的灰色代码块，长回复保持可读性
- **滚动按钮样式**：恢复了 `chat-new-messages` 类名，"新消息"滚动按钮使用现有的紧凑样式，不再渲染为全屏 SVG 覆盖层

## 重要问题修复

- **网关客户端**：未应答的网关 RPC 请求在有限超时后拒绝，并清除挂起状态，避免 stalled 连接永远泄漏挂起的 `GatewayClient.request()` 承诺
- **Ollama 推理可见性**：不再将原生 `thinking` 和 `reasoning` 字段提升到最终助手文本，避免本地推理模型泄露内部思考，感谢 @xi7ang
- **网关会话重置**：跨网关会话重置保留 `lastAccountId` 和 `lastThreadId`，`/reset` 后回复仍然正确路由到相同账户和线程，感谢 @Lanfei
- **macOS 开机自启**：避免自重启全新引导的 launchd 网关，给新安装更长时间变成健康状态，`openclaw onboard --install-daemon` 在慢速 Mac 和全新虚拟机快照不再误判失败
- **iMessage 远程附件**：在生成 SCP 之前拒绝不安全的远程附件路径，避免发送者控制的文件名注入 shell 元字符，感谢 @lintsinghua
- **Telegram Webhook**：在读取或解析请求体之前验证 Telegram webhook 密钥，未认证请求立即拒绝，感谢 @space08

## 安全更新

- **设备配对**：设置码改为单次使用，避免 pending 配对请求被静默重放并扩大到管理员权限，感谢 @tdjackey
- **执行审批**：在 macOS shell 链解析中将反斜杠换行视为 shell 行续，失败闭合防止检查跳过命令替换检查
- **插件安全**：禁用隐式工作区插件自动加载，克隆的仓库不能在没有明确信任决定的情况下执行工作区插件代码 (`GHSA-99qw-6mr3-36qr`)，感谢 @lintsinghua 和 @vincentkoc
