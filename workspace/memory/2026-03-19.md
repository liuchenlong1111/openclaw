# 2026-03-19 记忆

## OpenClaw Control UI 模型切换 Bug 分析

**时间**: 2026-03-19 00:28

**问题**: 用户在 OpenClaw Control UI 界面切换模型时，提示错误 `Failed to set model: GatewayRequestError: model not allowed: moonshot/glm-4-flash`

**根本原因**: 
- 用户的配置文件中 `glm-4-flash` 是在 `zhipu` provider 下定义的正确模型 ID 应该是 `zhipu/glm-4-flash`
- 但 UI 在切换模型时，把当前 session 的 provider (`moonshot`) 与选择的 model 名称拼接，导致生成了错误的 `moonshot/glm-4-flash`

**Bug 位置**: OpenClaw Control UI 的 Chat 页面模型选择器 (`index-UvgeZ3yV.js` 中 `data-chat-model-select` 相关逻辑)

**临时解决方案**: 使用命令 `/model zhipu/glm-4-flash` 直接切换，绕过 UI 的 bug

**用户配置确认无误**:
- `moonshot/kimi-k2.5` ✅
- `zhipu/glm-4-flash` ✅  
- `zhipu/glm-4-plus` ✅
